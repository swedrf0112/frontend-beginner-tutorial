# 文教品牌前端教學設計規格

## 目標
產出一份單一 Markdown 教學文件，讓完全沒有前端背景的讀者，也能用 `Node.js + React + Tailwind CSS` 做出一個四頁網站。成品風格以文教品牌官網為主要參考，重點是版型邏輯、資訊層級與正式感，而不是逐像素複製。

## 成品範圍
教學要帶出四個頁面：

1. 首頁
2. 教學品質頁
3. 報名流程頁
4. 教育活動列表頁

網站必須包含：

- `dark mode` 切換
- `zh / en` 中英文切換
- 透過私有 `npm` 套件接入公司內部 UI/icon 套件的示範

## 讀者假設
- 完全沒有 HTML / CSS / JavaScript / React 背景
- 可以照著終端步驟安裝套件與建立檔案
- 目標是先做出網站，再逐步理解元件、狀態、樣式與資料切分

## 教材寫法
整份教學固定使用同一節奏：

1. 先講這一步要做什麼
2. 明確指出要改哪個檔案
3. 一次只給一小段必要程式碼
4. 用白話解釋程式碼在做什麼
5. 說明完成後畫面應該看到什麼

不走「先講大量理論，再開始實作」的路線。

## 技術設計
- 建站方式：`Vite + React`
- 樣式：`Tailwind CSS`
- 路由：`react-router-dom`
- 狀態：以最小 `useState` + `localStorage` 管理語言與主題
- 資料：以本地 `content.js` 檔案保存中英文文案
- UI 套件：使用 `@company/ui`、`@company/icons` 作為代稱示範，優先拿來放 icon 與少量通用 UI

## 視覺策略
- 先用大框架建立文教品牌常見的版型節奏：頂部導覽、深色 hero、正式內容頁、流程卡片頁、活動列表頁
- 圖片在規劃與 mockup 階段可參考公開網站素材
- 正式教學不綁死外部圖片 URL，改寫成可替換本地素材流程

## 邊界
這份教學不做：

- 後端
- 真實搜尋
- CMS
- 真實活動篩選 API
- 活動詳情頁
- 公司內部套件真實名稱曝光

## 驗收標準
- 專案能啟動
- 四個路由都能進入
- `dark mode` 切換後四頁仍可讀
- 語言切換後主要文案能切成英文
- 至少一處 icon 明確來自公司內部 icon 套件
- 教育活動頁能呈現篩選列、活動標題、日期與 `SEE MORE`
